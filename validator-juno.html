Terms & Concepts

  • *Total Bytes Written: This node contains a juno|huahua copy. The HDD is monitored for write operations in order to learn the MTBF with TBW as the time passing variable.
  • ‚úù: Particular component, serialized by us. Details about each part will follow.
  • 16h: 16h aggregates (~10000 Blocks)
  • ToDo: I/O write errors. This will be helpful to know when we might need to discard the SSD. This should be kept at zero and the general recommendation is not to continue running the SSD in a node where the first IO writing failure occured; therefore this detector will be a good indicator of the end of life of the SSD in order to trigger a replacement.
  • Œ£ : Cumulative sum during the entire life of the applicable component.
  • üîß: Node under planned maintenance. 11.04.22: We did a full backup after the security patch in order to have a fallback (just in case).
